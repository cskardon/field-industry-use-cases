= Identification of Medicinal Products (IDMP)

== Introduction

The Identification of Medicinal Products (IDMP) ontology is a comprehensive framework for the global identification, description, and exchange of medicinal product information. Developed through the collaboration of the Pistoia Alliance and major pharmaceutical companies, the IDMP ontology codifies and structures data critical to regulatory compliance and patient safety. It is grounded in several ISO standards (including (https://www.iso.org/standard/70150.html[ISO 11615]), (https://www.iso.org/standard/70044.html[ISO 11616]), (https://www.iso.org/standard/69697.html[ISO 11238]), (https://www.iso.org/standard/81133.html[ISO 11239]), and (https://www.iso.org/standard/55033.html[ISO 11240])), providing a harmonized approach to capturing key attributes of substances, products, packaged products, and marketing authorizations. By releasing the IDMP ontology in RDF (Resource Description Framework) format, the Pistoia Alliance has enabled greater interoperability and integration across the pharmaceutical industry, regulatory agencies, and healthcare providers. The adoption of the IDMP ontology supports efficient regulatory submissions, robust pharmacovigilance, and more effective global traceability of medicinal products throughout their lifecycle.
The Identification of Medicinal Products ((https://spec.edmcouncil.org/idmp/ [IDMP])) is a system for identifying and tracking medicinal products. It is a global system that is used to identify and track medicinal products across the world.

== IDMP Data Model

=== Integrating IDMP RDF in Neo4j

The IDMP ontology, provided in Resource Description Framework (RDF) format, can be integrated into Neo4j to take advantage of the graph model for querying and analysis. There are two primary approaches for importing and working with RDF/OWL data such as the IDMP ontology in Neo4j:

**Neo4j Neosemantics (n10s) Plugin**  
Neo4j's (https://neo4j.com/labs/neosemantics[Neosemantics (n10s)]) plugin enables the import and mapping of RDF data (including ontologies and instance data) directly into Neo4j. With n10s, you can:

- Import RDF (Turtle, RDF/XML, N-Triples, JSON-LD) via Cypher procedures
- Map RDF schema concepts (classes, properties) to labeled property graph structures
- Query data using Cypher enriched with RDF semantics

Documentation (https://neo4j.com/labs/neosemantics[here])

**RDFLib-neo4j Python Library**  
   (https://github.com/uchida/rdflib-neo4j[RDFLib-neo4j]) (https://github.com/neo4j-labs/rdflib-neo4j[GitHub Repository]) enables integration between (https://rdflib.readthedocs.io/[RDFLib]) (a popular RDF processing library in Python) and Neo4j. This provides another route to transfer and query IDMP RDF data in Neo4j.

Documentation (https://rdflib.readthedocs.io/en/stable/[here])

=== IDMP Layers

There are 4 layers to the IDMP data model:

1. Substance Layer
2. Product Layer
3. Packaged Product Layer
4. Marketing Authorization Layer

Each layer has a set of nodes and relationships that are used to represent the data in the layer.

==== Substance Layer

Substance nodes represent the active substance of a pharmaceutical product.

Node:
Substance
	•	name: Acetylsalicylic Acid
	•	inn: aspirin
	•	molecular_formula: C9H8O4
	•	cas: 50-78-2

Relationship:
Substance
→ HAS_MODIFICATION →

Node:
SubstanceModification
	•	type: acid
	•	parent_substance: salicylic acid

⸻

==== Product Layer

PharmaceuticalProduct nodes represent the pharmaceutical product, which is a combination of a substance and a dose form.

Node:
PharmaceuticalProduct
	•	name: Aspirin 500mg Tablet
	•	product_id: EU/1/…

Relationships:

PharmaceuticalProduct
→ CONTAINS (strength: 500mg, strength_type: equal) →
Substance (Acetylsalicylic Acid)

PharmaceuticalProduct
→ HAS_DOSE_FORM →
DoseForm
	•	term: film-coated tablet
	•	edqm_code: 10219000

PharmaceuticalProduct
→ HAS_ROUTE →
RouteOfAdministration
	•	term: oral use
	•	edqm_code: 20053000

PharmaceuticalProduct
→ MANUFACTURED_BY (role: finished_product) →
Organisation
	•	name: Bayer Manufacturing GmbH

⸻
==== Packaging & Authorization Layer

PackagedProduct nodes represent the packaged product, which is a combination of a pharmaceutical product and a pack size.

Node:
PackagedProduct
	•	pack_size: 20
	•	barcode: 4007971013704

Relationship:
PackagedProduct
→ IS_PRESENTATION_OF →
PharmaceuticalProduct

⸻
==== Marketing Authorization Layer

MarketingAuthorization nodes represent the marketing authorization for a packaged product.

Node:
MarketAuthorization
	•	auth_number: DE-12345
	•	status: active
	•	expiry: 2027-06-01

Relationships:

MarketAuthorization
→ AUTHORIZES → PackagedProduct

MarketAuthorization
→ GRANTED_BY →
RegulatoryAgency
	•	name: BfArM
	•	country: DE

MarketAuthorization
→ HELD_BY →
Organisation
	•	name: Bayer AG
	•	country: DE

ActiveSubstanceManufacturing nodes represent the manufacturing of an active substance.

Substance (Acetylsalicylic Acid)
→ MANUFACTURED_BY (role: active_substance) →
Organisation
	•	name: Bayer Active Science
